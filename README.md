# Citywide Stormwater Catchment Delineation and Pipe Capacity Analysis

**Kaitlyn Rawlings** | Asset Management GIS Analyst  
ArcGIS Pro Model Builder · Python (arcpy) · Manning's Equation · NRCS TR-55

---

This repository contains the generalized Python scripts and workflow documentation for the **Citywide Stormwater Catchment Delineation and Pipe Capacity Analysis** project, originally developed for the City of Lenexa, Kansas Public Works department in April 2026. The project delineates stormwater catchment areas citywide and examines pipe capacity against estimated runoff to identify potential flood risk locations.

The full project, including maps, results, and methodology, is documented in the [portfolio project page](https://kaitlyn-rawlings.github.io/kaitlynrawlings.github.io/projects/stormwater-catchment.html).

Using a 1-meter resolution DEM and the D8 flow routing algorithm (O'Callaghan & Mark, 1984), terrain analysis characterizes surface flow direction and accumulation, which are used to delineate catchment polygons from storm inlet structure locations. A custom Python script conducts a pipe capacity analysis for each catchment using Manning's equation (Chaudhry, 2008; ASCE, 1992). A second script estimates runoff volume per catchment using NRCS curve numbers derived from NLCD land cover and SSURGO hydrologic soil groups, applying the NRCS TR-55 method (USDA NRCS, 1986) with a 10-year 24-hour design storm. Catchments where runoff exceeds pipe capacity are identified as potential flood risk locations. Open channel drainage features and flood control detention facilities present within each catchment are also inventoried to provide broader infrastructure context.

The workflow is designed to be transferable to any municipality with GIS infrastructure data and a high-resolution DEM.

## Workflow

The analysis was conducted in four phases, automated in ArcGIS Pro Model Builder. Phases 3 and 4 are implemented as standalone Python scripts run after the ModelBuilder outputs are complete.

**Phase 1 — Terrain Flow Analysis**  
The 1-meter DEM is processed through Fill, Flow Direction (D8), and Flow Accumulation to establish how water moves across the landscape. A stream network is extracted where flow accumulation exceeds a defined threshold, and a depression depth raster is generated by subtracting the original DEM from the filled surface to identify locations where surface ponding is likely to occur.

**Phase 2 — Catchment Delineation**  
Storm inlet structures serve as pour points. The Snap Pour Point tool moves each inlet to the nearest high-accumulation cell to account for spatial offsets between structure locations and the DEM. The Watershed tool delineates the contributing drainage area for each inlet, and the output is converted to vector polygons and dissolved by structure number.

**Phase 3 — Pipe Capacity Analysis (`catchment_capacity.py`)**  
Manning's equation is applied to each enclosed gravity pipe connected to a catchment's inlet structure. Capacity values are summed per catchment and normalized by area. Where pipe slope data is null, a configurable default slope is applied per local stormwater design standards.

**Phase 4 — Runoff and Risk Assessment (`catchment_runoff.py`)**  
A curve number raster is constructed by combining NLCD land cover with SSURGO hydrologic soil groups and applying the TR-55 CN lookup table. Zonal Statistics produces an area-weighted average CN per catchment. The TR-55 rainfall-runoff equation is applied to estimate runoff volume for the design storm, and catchments where runoff exceeds pipe capacity are flagged as at-risk. An adjusted risk flag accounts for the presence of open channel conveyance and flood control BMPs within at-risk catchments.

## Scripts

Both scripts require **ArcGIS Pro** with the **Spatial Analyst** extension and the **arcpy** Python library. Field names in the `CONFIGURATION` block at the top of each script are placeholders and must be updated to match your asset management system's schema before running.

### `catchment_capacity.py`
Joins each catchment to its connected enclosed gravity pipes via upstream/downstream structure number fields. Applies Manning's full-pipe equation to each pipe and sums capacity per catchment.

**Output fields added to catchment feature class:**

| Field | Type | Description |
|---|---|---|
| `PIPE_COUNT` | Short | Number of enclosed gravity pipes connected to inlet |
| `TOTAL_CAP_AF` | Double | Total full-flow pipe capacity, acre-feet per 24-hour storm |
| `TOTAL_CAP_GL` | Double | Total full-flow pipe capacity, gallons |
| `DOM_DIA_IN` | Short | Dominant (most common) pipe diameter, inches |
| `AVG_SLOPE_PCT` | Double | Average slope across connected pipes, percent |
| `CAP_PER_ACRE` | Double | TOTAL_CAP_AF divided by catchment area in acres |

### `catchment_runoff.py`
Builds a CN raster from NLCD and SSURGO inputs, calculates area-weighted average CN per catchment using Zonal Statistics, and applies the NRCS TR-55 equation to estimate runoff volume for a user-defined design storm. Conducts spatial intersects against open channel conduits and BMP assets to inventory supplemental drainage infrastructure.

**Output fields added to catchment feature class:**

| Field | Type | Description |
|---|---|---|
| `AVG_CN` | Double | Area-weighted average NRCS curve number |
| `RUNOFF_IN` | Double | Estimated runoff depth, inches |
| `RUNOFF_AF` | Double | Estimated runoff volume, acre-feet |
| `CAP_DEFICIT_AF` | Double | Runoff volume minus pipe capacity (positive = deficit) |
| `RISK_FLAG` | Short | Pipe-only risk: 1=at-risk, 0=adequate, -1=no pipe data |
| `DITCH_FT` | Double | Linear feet of ditch within catchment |
| `SWALE_FT` | Double | Linear feet of swale within catchment |
| `STREAM_FT` | Double | Linear feet of stream channel within catchment |
| `OPEN_CH_FT` | Double | Total open channel footage (all types) |
| `BMP_PRESENT` | Short | 1 if active flood control BMP intersects catchment |
| `BMP_TYPE` | Text | Comma-separated list of BMP type names present |
| `MAX_POND_IN` | Double | Maximum depression depth, inches |
| `AVG_POND_IN` | Double | Average depression depth, inches |
| `RISK_FLAG_ADJ` | Short | Adjusted risk accounting for open channel and BMP presence |

**RISK_FLAG_ADJ values:**

| Value | Meaning |
|---|---|
| `1` | At-risk, no supplemental infrastructure present |
| `2` | At-risk, open channel conveyance present — field verify |
| `3` | At-risk, flood control BMP present — field verify |
| `4` | At-risk, both open channel and BMP present — field verify |
| `0` | Adequate pipe capacity |
| `-1` | No pipe data, no open channel or BMP present |
| `-2` | No pipe data, but open channel or BMP infrastructure present |

## Adapting for Your Municipality

To use these scripts for a different study area:

1. Update the `GDB` path and all input feature class names in the `CONFIGURATION` block
2. Update field name mappings to match your asset management system's schema
3. Update `DESIGN_STORM_P` to the NOAA Atlas 14 point frequency estimate for your study area and desired return interval
4. Update `DEFAULT_SLOPE_PCT` to your jurisdiction's stormwater design standard minimum slope
5. Update Manning's n values in `MANNINGS_N` if your pipe material codes differ
6. Update `FLOOD_CONTROL_BMP_TYPES` and `BMP_TYPE_NAMES` to match your BMP asset classification system

## Limitations

The D8 flow routing algorithm can produce unrealistically large catchment areas on flat terrain. Manning's equation represents an optimistic upper bound on pipe capacity, assuming full-flow conditions free of blockages. Open channel and BMP hydraulic capacity are inventoried but not computed. See the full project documentation for a complete discussion.
